プロダクト仕様書 (PRD): Airbnb AI Host Assistant

バージョン: 1.0 (Draft)
プラットフォーム: Google Chrome Extension (Manifest V3)
更新日: 2023-10-27

1. プロダクト概要

1.1 コンセプト

Airbnbの物件詳細ページを閲覧するだけで、サイドバーに常駐するAIが「地域相場」「競合データ」「AirDNA等の外部データ」を統合し、**「今の価格設定の妥当性」と「収益向上のための具体的な改善策」**を自然言語でアドバイスするChrome拡張機能。

1.2 解決する課題

データの断絶: ホストはAirbnbの画面と、AirDNAなどの分析ツールを行き来しており、情報の突き合わせが面倒。

分析の属人化: データを見ても「結局いくらに設定すべきか」「文章をどう直すべきか」の判断が難しい。

時間コスト: 競合調査や説明文の推敲に多くの時間を費やしている。

1.3 ターゲットユーザー

個人・副業ホスト: 1〜3物件を運用。データ分析よりゲスト対応を優先したい層。

小規模運営代行（〜10室）: 効率的に複数物件のパフォーマンスを底上げしたい事業者。

AirDNA契約者: データを持て余しており、具体的なアクションプランを求めている層。

2. システムアーキテクチャ

2.1 構成図（概念）

graph LR
    A[User / Chrome Browser] -- 閲覧 --> B(Airbnb Listing Page)
    A -- 操作 --> C(Chrome Extension SidePanel)
    C -- DOM解析 --> B
    C -- API Request --> D{Backend / Cloud Functions}
    D -- Fetch --> E[AirDNA API / External Data]
    D -- Prompt + Data --> F[LLM (OpenAI/Claude)]
    F -- Response --> D
    D -- Actionable Advice --> C


2.2 テクニカルスタック (推奨)

Frontend: React, TypeScript, Tailwind CSS (Chrome Extension Side Panel API)

Backend: Node.js (Firebase Functions または AWS Lambda)

Database: Firestore (ユーザー設定、キャッシュ、分析履歴保存)

AI Model: GPT-4o または Claude 3.5 Sonnet (複雑な推論と自然な日本語生成のため)

Data Source: AirDNA API (またはスクレイピングデータ + 独自DB)

3. 機能要件 (MVPスコープ)

3.1 Airbnbページ認識・データ抽出 (Context Awareness)

トリガー: URLが airbnb.com/rooms/{id} または airbnb.jp/rooms/{id} に一致した場合にアクティブ化。

DOM抽出データ:

物件ID (room_id)

物件タイトル

ホスト名・スーパーホストフラグ

宿泊料金 (Base Price / Cleaning Fee)

最大定員・ベッド数・バスルーム数

位置情報 (Lat/Lon - ページ内メタデータより抽出)

レビュー平均点・件数

説明文 (Description) ・アメニティリスト

3.2 地域相場ダッシュボード (Market Data)

サイドバー上部に重要指標を固定表示する。

表示項目:

エリアADR (平均客単価): 類似物件（同エリア・同定員）の過去30日/今後30日の平均。

エリア稼働率 (Occupancy Rate): エリア全体の予約埋まり具合。

RevPAR (販売可能な客室1室あたりの収益): 物件の稼ぐ力を示す指標。

データソース: AirDNA API等の外部データ、またはDOMから取得した周辺物件データの集計値。

3.3 AI価格コンサルタント (Smart Pricing)

「いくらにすべきか」に対する根拠ある回答を提示する。

入力: 現在設定価格 vs エリアADR、季節係数、直近のイベント情報。

出力UI:

判定バッジ: 「適正」「割高（機会損失の恐れ）」「割安（収益最大化の余地あり）」

推奨価格レンジ: 平日 / 週末 / 直前予約 の3パターンで提示。

アドバイス理由: 「来週はエリア全体の稼働率が90%を超えているため、強気の価格設定（+15%）が推奨されます」等の自然言語解説。

3.4 物件改善アドバイザー (Listing Optimizer)

クリック率（CTR）と成約率（CVR）向上を支援する。

タイトル改善:

ターゲット（ファミリー、リモートワーク、カップル）別に3案を提示。

SEOキーワード（駅近、Wifi完備など）の盛り込み提案。

説明文リライト:

「魅力的な冒頭文」の生成。

ネガティブレビュー対策（例：「騒音が気になる」というレビューがある場合、「静かな住宅街ではありませんが、活気ある繁華街の中心です」等のポジティブ変換）。

アメニティ推奨:

競合にあって自物件にない設備（例：ネスプレッソ、ワークチェア）を指摘。

3.5 競合比較ビュー (Comp Set)

対象: 同エリア・同価格帯・同定員の上位3〜5件。

比較項目: 価格、レビュースコア、写真の枚数、主要アメニティ。

AI分析: 「あなたの物件は競合Aより¥2,000高いですが、レビュー数が少ないため苦戦する可能性があります。写真を増やして差別化しましょう」といった総評。

4. 拡張機能・将来構想 (Phase 2以降)

4.1 マルチ物件管理 & ダッシュボード

拡張機能のポップアップ外（フルページタブ）で、登録全物件の「AIスコア」を一覧表示。

物件ごとの課題（価格調整が必要、レビュー返信が必要など）をTODOリスト化。

4.2 レポート出力

「月次運用レポート」をPDF/Markdownでワンクリック生成。

オーナーへの報告業務を効率化したい運営代行会社向け機能。

4.3 アラート通知 (Background Worker)

ブラウザを閉じていてもバックグラウンドでデータを監視。

「周辺の予約が急増しています（イベント検知）。価格を上げてください」と通知を送る。

4.4 写真AI解析 (Computer Vision)

GPT-4o Vision等のAPIを利用。

「トップ画像が暗い」「水回りの写真がない」などの具体的な改善点を指摘。

クリックされやすい写真の並び順を提案。

5. UX/UI ガイドライン

5.1 サイドバー (Side Panel)

幅: 可変（デフォルト400px）。

トーン & マナー: AirbnbのUIに馴染むクリーンなデザイン（白背景、Airbnbに近いフォント、アクセントカラーは信頼感のある青またはAirbnbのラッシュ）。

インタラクション:

AI生成中はスケルトンスクリーンまたはローディングアニメーションを表示。

提案されたタイトルや説明文には「コピー」ボタンまたは「Airbnb編集画面へ反映（技術的に可能な場合）」ボタンを配置。

5.2 設定画面

AirDNA / OpenAI APIキーの入力フォーム（暗号化して保存）。

分析対象エリアの半径設定（例：500m, 1km）。

言語設定（日本語/英語/中国語）。

6. データフローとセキュリティ

6.1 データフロー

Client: Content ScriptがDOM情報を取得 → Background Scriptへ送信。

Background: 外部API（AirDNA等）から市場データを取得。

Processing: 物件データ + 市場データを整形し、LLMへプロンプトとして送信。

Output: LLMの回答をパースし、サイドバーUIへレンダリング。

6.2 セキュリティ要件

APIキー管理: クライアントサイドにAPIキーをハードコードしない。ユーザー入力キーは chrome.storage.local に保存し、外部通信は原則として自身のBackend Proxy経由で行うことを推奨（APIキー流出防止）。

個人情報: ゲストの氏名や詳細な住所など、分析に不要なPII（個人特定情報）はバックエンドに送信しない。
